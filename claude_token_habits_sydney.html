When Claude gets something wrong, it's tempting to type "No, I meant…" and send again. Don't. Every follow-up stacks onto the history, and Claude re-reads all of it every single turn.

Cost formula: S × N(N+1) / 2 (S = avg tokens/exchange, N = number of messages) At ~500 tokens per exchange: 5 messages → 7,500 tokens 10 messages → 27,500 tokens 20 messages → 105,000 tokens 30 messages → 232,000 tokens ← 31× more than msg 1

Instead: hover over your original message → click Edit → fix it → regenerate. The old exchange is replaced, not stacked.

↩ Edit first. Always.

A 100-message chat at 500 tokens each burns over 2.5 million tokens — the vast majority of that is just re-reading old history on loop.

⚠️ One developer tracked his usage and found 98.5% of tokens went to re-reading history. Only 1.5% produced actual output.

When a chat gets long: ask Claude to summarise everything → copy it → open a new chat → paste it as your first message. Full context, fresh headroom.

↺ Summarise → new chat → paste

Three separate messages = three full context loads. One message with three tasks = one context load. You save tokens twice: fewer reloads, and you stay further from your limit.

❌ "Summarise this article"
❌ "Now list the main points"
❌ "Now suggest a headline"

✅ "Summarise this article, list the main points, and suggest a headline."

Bonus: Claude often gives better answers when it sees the full picture upfront.

≡ Three questions. One prompt. Always.

Every time you drag a PDF into a new chat, Claude re-tokenises the whole thing from scratch. Use Projects instead — upload once, and every conversation inside that project references it without burning tokens again.

💡 Great for: contracts, style guides, company briefs, long reference docs. This single habit can dramatically cut your token spend.
⊕ Upload once. Reference forever.

Without saved context, every chat wastes 3–5 messages just setting Claude up: "I'm a marketer, I write casually, short paragraphs, Australian spelling…"

Go to Settings → Memory & User Preferences. Write your role, tone, and working style once. Claude applies it automatically to every new chat — no setup needed.

⚙ Settings → Memory → Set it once

Web search, connectors, and Extended Thinking all add tokens to every response — even when you don't need them. If you're writing, editing, or brainstorming, there's no reason to have search running.

⚙️ Simple rule: if you didn't deliberately turn a feature on, turn it off. Only enable Extended Thinking if your first attempt fell short.
✕ Off by default. On by intention.

Haiku handles grammar checks, quick translations, brainstorming, and formatting at a fraction of the cost of Sonnet or Opus. Saving your big-model budget for big tasks is the highest-leverage habit on this list.

Haiku → drafts, edits, quick answers (low cost) Sonnet → real work, analysis, coding (mid cost) Opus → deep research, hard problems (high cost)

Using Haiku for simple tasks can free up 50–70% of your budget for things that actually need power.

Claude uses a rolling 5-hour window — not a midnight reset. Tokens you used at 9am stop counting toward your limit by 2pm. Burning your entire allowance in one morning session wastes most of your daily capacity.

🇦🇺 Sydney tip: Try splitting into three sessions — morning (~8–10am), a midday break (~1–2pm), and an evening wind-down (~7–9pm). Each session benefits from fresh headroom as old usage rolls off.
◷ Morning → Midday → Evening
